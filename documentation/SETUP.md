# Setup Guide

Get the School Management & Timetable System running locally in about 10 minutes.

---

## Prerequisites

| Tool       | Version          | Notes                                      |
| ---------- | ---------------- | ------------------------------------------ |
| Python     | 3.10+ (3.13 OK)  | For the Flask backend                      |
| Node.js    | 18+              | For the React frontend                     |
| npm        | 9+               | Comes with Node.js                         |
| SQLite     | bundled w/ Python| No separate install needed                 |
| Docker     | 24+ (optional)   | Only if you want to run via `docker-compose`|

---

## 1. Clone & Enter the Project

```bash
git clone <your-repo-url> "cpp project"
cd "cpp project"
```

---

## 2. Backend Setup

### a. Create a virtual environment and install dependencies

```bash
cd backend
python3 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

Installed packages (see `backend/requirements.txt`):
`Flask`, `Flask-CORS`, `Flask-SQLAlchemy`, `python-dotenv`, `gunicorn`, `psycopg2-binary`, `PyJWT`, `reportlab`, `WeasyPrint`, `Pillow`.

### b. Seed the database

Choose **one**:

```bash
# Minimal seed (a few sample records)
python seed.py

# Full realistic dataset (recommended) — 2,800 students, 75 teachers, etc.
python seed_realistic.py
```

This creates `timetable.db` at the project root and auto-generates user accounts (see Test Credentials below).

### c. Start the API

```bash
flask run --port=5000
# or
python app.py
```

API base URL: **http://localhost:5000/api**

Verify with:

```bash
curl http://localhost:5000/api/health
# → {"status": "ok"}
```

---

## 3. Frontend Setup

In a **new terminal**:

```bash
cd frontend
npm install
npm start
```

The app opens automatically at **http://localhost:3000** and proxies API calls to the backend on port 5000.

To point the frontend at a different backend, set `REACT_APP_API_URL` in `frontend/.env`:

```env
REACT_APP_API_URL=http://localhost:5000/api
```

---

## 5. Run with Docker (optional)

The repo ships with a `docker-compose.yml` that builds both services:

```bash
docker compose up --build
```

This brings up:
- `backend` → http://localhost:5000
- `frontend` → http://localhost:3000

Stop with `Ctrl+C`, clean up with `docker compose down -v`.

## Common Tasks

### Reset the database
```bash
rm timetable.db
cd backend && python seed_realistic.py
```

### Inspect the database
```bash
sqlite3 timetable.db
.tables
SELECT COUNT(*) FROM students;     # → 2800
SELECT class_grade, COUNT(*) FROM students GROUP BY class_grade;
.quit
```

### Run the backend in production mode
```bash
cd backend
gunicorn -w 4 -b 0.0.0.0:5000 app:app
```

### Build the frontend for production
```bash
cd frontend
npm run build
# Static bundle is in frontend/build/
```

## Troubleshooting

**Backend won't start / import errors**
```bash
cd backend
source venv/bin/activate
pip install --upgrade -r requirements.txt
```

**`401 Unauthorized` from the frontend**
- Make sure you ran a seed script so user accounts exist.
- Verify the token is being sent: open DevTools → Network → request headers should include `Authorization: Bearer …`.
- Clear stale tokens: `localStorage.clear()` in the browser console, then log in again.

**Database is empty / login fails**
```bash
rm timetable.db
cd backend && python seed_realistic.py
```

**Frontend won't compile**
```bash
cd frontend
rm -rf node_modules package-lock.json
npm install
```

**Port already in use**
- Backend: `flask run --port=5001` (then update `REACT_APP_API_URL`).
- Frontend: `PORT=3001 npm start`.

**CORS errors**
- Confirm the backend is running on the port the frontend expects.
- `Flask-CORS` is enabled by default in `app.py`; if you've customized it, allow `http://localhost:3000`.
